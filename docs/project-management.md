# Organización del proyecto

## Introducción

Para gestionar el desarrollo del proyecto se ha utilizado un tablero de Trello siguiendo una metodología basada en Kanban.

El objetivo es organizar las tareas, visualizar el progreso y facilitar la planificación del trabajo.


## Estructura del tablero

El tablero está dividido en las siguientes columnas:

- Backlog → tareas pendientes de definir o priorizar  
- To Do → tareas listas para empezar  
- In Progress → tareas en desarrollo  
- Review → tareas en revisión o pruebas  
- Done → tareas completadas  

## Gestión de tareas

Cada funcionalidad del proyecto se ha representado mediante una tarjeta en Trello.

Ejemplos de funcionalidades:

- 9. implementar navegacion entre paginas  
- 12. documentar la capa de red y el contrato de tipos en docs/api-client.md
- 6. crear componentes reutilizables usando React

## Flujo de trabajo

Las tareas se mueven entre columnas según su estado:

1. Se añaden al Backlog  
2. Se pasan a To Do cuando están listas  
3. Se mueven a In Progress durante el desarrollo  
4. Se revisan en Review  
5. Se marcan como Done al completarse  


## Conclusión

El uso de Trello y la organización en columnas ha permitido tener una visión clara del estado del proyecto y avanzar de forma estructurada durante el desarrollo.
